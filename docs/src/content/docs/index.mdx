Working with kkrpc in an AI-powered editor like Claude Code? Add these skills to get intelligent code assistance:

```bash
# Copy kkrpc skills to your global Claude Code skills folder
cp -r skills/kkrpc ~/.claude/skills/
cp -r skills/interop ~/.claude/skills/
```

This provides your AI assistant with:

- **kkrpc skill**: How to use kkrpc in TypeScript projects
- **interop skill**: How to implement kkrpc clients/servers in other languages

See the [`skills/`](https://github.com/kunkunsh/kkrpc/tree/main/skills) directory in the GitHub repo for details.

## Features

<CardGrid stagger>
	<Card title="Property Access">
		Remote property getters and setters with dot notation (`await api.prop`, `api.prop = value`)
	</Card>
	<Card title="Enhanced Error Preservation">
		Complete error object preservation across RPC boundaries including stack traces and custom
		properties
	</Card>
	<Card title="HTTP">Call API over HTTP like calling local functions (similar to tRPC)</Card>
	<Card title="WebSocket">Call API over WebSocket (with callback support and bidirectional)</Card>
	<Card title="Web Worker">Bidirectional communication between Web Worker and main thread</Card>
	<Card title="iframe">Bidirectional communication between iframe and parent window</Card>
	<Card title="stdio">
		IPC over stdio between JavaScript/TypeScript processes (e.g. Node.js/Deno/Bun)
	</Card>
	<Card title="Chrome Extension">Communicate between content script and background script</Card>
	<Card title="Tauri">Call functions in bun/node/deno processes from Tauri app with JS/TS</Card>
	<Card title="RabbitMQ">Reliable message queue communication with AMQP protocol</Card>
	<Card title="Redis Streams">High-performance stream-based messaging with persistence</Card>
	<Card title="Kafka">Distributed streaming with high throughput and fault tolerance</Card>
</CardGrid>
